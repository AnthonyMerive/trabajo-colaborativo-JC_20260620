Acerca

Esta página es una plantilla para una clase práctica de GitHub. Cada participante puede subir cambios a home.html, abrir un pull request y ver cómo se realiza el merge.

GitHub Pages

Para publicar este repositorio en GitHub Pages, activa Pages en la rama main y usa la raíz del repositorio.

Este repositorio incluye un index.html que redirige automáticamente a home.html, para que tu página sea visible desde la URL de Pages.

Workflow de merge

  1. Clona el repositorio.
  2. Actualiza main y crea una rama nueva.
  3. Modifica home.html con tu contribución.
  4. Haz commit y sube tu rama al remoto.
  5. Abre un pull request y pide revisión.
  6. Haz merge una vez aprobado.

Actividad colaborativa

Cada persona debe hacer cambios en home.html y documentar brevemente qué aportó. Algunas ideas:

  • Añadir una breve explicación sobre Git o GitHub.
  • Escribir un consejo para resolver conflictos.
  • Describir cómo se revisa un pull request.

Instrucciones

Usa estos comandos para trabajar en Git:

git checkout main
git pull origin main
git checkout -b mi-rama-nombre
# edita home.html
git add home.html
git commit -m "Añade mi contribución a home.html"
git push -u origin mi-rama-nombre

Luego crea un pull request en GitHub y describe tus cambios.

Al hacer merge, tu contribución aparecerá en el sitio de GitHub Pages.
