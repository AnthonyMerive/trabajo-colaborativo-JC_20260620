🌐 GitHub Pages - Publicación

Para publicar este repositorio en GitHub Pages:

  1. Ve a Settings → Pages
  2. Selecciona la rama main
  3. Elige la carpeta raíz del repositorio
  4. GitHub automáticamente publicará el sitio
💡 Nota: El archivo index.html redirige a home.html, así tu página está visible desde la URL de GitHub Pages.

🔄 Workflow de merge

Sigue estos pasos para contribuir:

  1. Clona el repositorio en tu máquina
  2. Actualiza main con git pull origin main
  3. Crea rama nueva con un nombre descriptivo
  4. Modifica este archivo con tu contribución
  5. Commit tus cambios con un mensaje claro
  6. Push tu rama al repositorio remoto
  7. Abre PR desde tu rama hacia main
  8. Merge una vez que sea aprobado
✓ Consejo: Usa ramas separadas para que cada cambio sea visible en el historial de Git.

⚙️ Instrucciones de Configuración

1. Clonar el repositorio

git clone https://github.com/AnthonyMerive/trabajo-colaborativo-JC_20260620.git
cd trabajo-colaborativo-JC_20260620

2. Crear tu rama

git checkout main
git pull origin main
git checkout -b tu-nombre-rama

3. Hacer cambios

Edita este archivo home.html y agrega tu sección o modifica el historial.

4. Subir cambios

git add home.html
git commit -m "Agregué: descripción de tu cambio"
git push -u origin tu-nombre-rama

5. Abrir Pull Request

Ve a GitHub y abre un PR desde tu rama hacia main. Describe qué cambios hiciste y por qué.

⚠️ Recuerda: Trabaja en ramas separadas para que cada cambio se vea en el historial de GitHub y para poder practicar merges reales. Siempre trabaja en ramas separadas y nunca hagas commit directamente en main.
